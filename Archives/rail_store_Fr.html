Documentation du modèle

Ce guide explique chaque paramètre, sa signification physique, comment le renseigner, et les formules de calcul sous-jacentes. Utiliser conjointement avec les onglets Inputs et Assumptions.

📋 Workflow — Comment utiliser le modèle
  1. Inputs — Paramètres projet : Renseigner la surface atelier brute, la hauteur utile, la taille de flotte. Ces données initient la chaîne de calcul.
  2. Inputs — Familles de stockage : Ajuster les parts (%) par famille. Utiliser les boutons de profil système (Metro, Tram…) pour charger des valeurs typiques, puis affiner. La somme doit faire 100%.
  3. Assumptions : Vérifier les paramètres géométriques et d'efficacité. Les valeurs par défaut sont calibrées pour un dépôt Metro de taille moyenne. Modifier uniquement ce qui diffère du projet réel.
  4. Results : Lire les capacités recommandées par famille et les totaux par type d'équipement (travées, modules, ML, places). Ces valeurs sont les quantités installées à spécifier en préconception.
  5. Layout 2D : Ajuster la largeur du bâtiment magasin. Glisser et redimensionner les zones pour tester différentes configurations spatiales. Vérifier le ratio empreinte / S_net.
🔢 Chaîne de calcul — Formules
Étape 1 — Surface nette de stockage (S_net)
▶
Principe

La surface totale de l'atelier est réduite en deux étapes : d'abord on extrait la part réservée au magasin (k_store), puis on retire les zones non-stockage à l'intérieur de ce magasin (k_nostor). Le reste est la surface nette disponible pour les racks.

Valeurs typiques

k_store = 18-22% pour un dépôt ferroviaire standard.
k_nostor = 30-40% (réception 8%, staging 10%, quai 5%, bureau/SST 5%, circulation 10-15%).

── Surface brute du magasin ────────────────────────── S_gross = S_workshop × k_store = 6 000 m² × 0.18 = 1 080 m² ── Zone non-stockage (réception, staging, bureau…) ─── S_nostor = S_gross × k_nostor = 1 080 × 0.35 = 378 m² ── Surface nette de stockage ← BASE DE TOUS LES CALCULS S_net = S_gross − S_nostor = 1 080 − 378 = 702 m² Forme compacte : S_net = S_workshop × k_store × (1 − k_nostor)
Étape 2 — Capacité par famille de stockage
▶
Principe

S_net est découpé selon les parts (%) de chaque famille. Chaque zone brute est réduite par un facteur d'efficacité de layout (eff) pour obtenir l'empreinte réelle des équipements (aisles déduits). L'empreinte est ensuite convertie en unités de capacité selon le type de stockage.

── Pour chaque famille i ────────────────────────────── S_raw_i = S_net × share_i ← zone brute allouée à la famille = 702 × 0.30 = 210.6 m² (ex. palettes 30%) S_fp_i = S_raw_i × eff_i ← empreinte rack après déduction aisles = 210.6 × 0.45 = 94.8 m² (eff_pallet = 45%) ── Conversion S_fp → capacité (par type) ───────────── Racks palettes : N_pos = S_fp / a_pos × N_lev = 94.8 / 1.1925 × 3 = 238 positions Rayonnage : ML_sol = S_fp / she_D = 70.2 / 0.50 = 140.4 m linéaires Cantilever : ML = S_fp / (2 × can_a) = 12.3 / 1.60 = 7.7 m linéaires Stockage sol : places = S_fp / (flo_W × flo_D) = 45.6 / 4.00 = 11.4 places Dédié/Répar : zone = S_fp (m²) ── Application marge croissance ────────────────────── Rec. installé = CEILING ( capacité_brute × (1 + k_gr), granulométrie ) ex. : CEILING ( 238 × 1.10, 1 ) = 262 positions Granulométrie : 1 pour positions et places · 0.5 pour ML (arrondi au demi-mètre)
Étape 3 — Géométrie rack palette (bay_W, bay_D, a_pos, N_lev)
▶
── Dimensions d'une travée ──────────────────────────── bay_W = 2×pal_w + 2×clr_s + clr_b ← largeur totale, 2 palettes côte à côte = 2×1.20 + 2×0.075 + 0.10 = 2.65 m bay_D = pal_d + clr_d ← profondeur rack (pallet + tolérance) = 0.80 + 0.10 = 0.90 m a_pos = bay_W × bay_D / pal_bay ← empreinte par emplacement palette (m²/pos) = 2.65 × 0.90 / 2 = 1.1925 m²/pos ── Calcul du nombre de niveaux (auto depuis H_useful) ─ h_lev = pal_h + clr_v ← hauteur par niveau (charge + dégagement) = 1.20 + 0.20 = 1.40 m N_lev = INT( (H_useful − clr_bt − clr_tp) / h_lev ) = INT( (6.00 − 0.15 − 0.30) / 1.40 ) = INT( 3.96 ) = 3 niveaux h_frame = clr_bt + N_lev × h_lev + clr_tp ← vérification vs H_useful = 0.15 + 3×1.40 + 0.30 = 4.65 m (doit être ≤ H_useful)
Étape 4 — Facteur d'efficacité de layout (eff)
▶
Définition

Le facteur eff représente la fraction de la zone brute (S_raw) qui se transforme en empreinte réelle de rack, une fois les allées principales et allées transversales déduites. C'est un paramètre empirique issu de l'expérience de layouts réels.

Exemple : zone racks palettes, eff_pal = 0.45 S_fp = S_raw × eff_pal = 210.6 × 0.45 = 94.8 m² La zone brute est 210.6 m². Sur ces 210.6 m² : · 94.8 m² sont occupés par les racks (empreinte réelle) · 115.8 m² sont les allées de manutention (= 55% de la zone) Valeurs typiques par type de stockage : eff_pal = 0.45 ← allées chariot élévateur larges (3-3.5m) eff_she = 0.50 ← allées piéton plus étroites (0.8-1.0m) eff_can = 0.35 ← allées larges pour grue/pont, items longs eff_flo = 0.65 ← emplacements délimités, circulation directe

Distribution des familles de stockage
▶
Principe des parts (%)

Chaque famille reçoit une fraction de S_net. La somme des 8 parts doit être exactement 100%. Ces parts reflètent le poids volumétrique relatif de chaque catégorie de pièces dans la politique de stock du dépôt.

Taux d'occupation cibles (occ)
▶
Rôle dans le calcul

Le taux d'occupation cible définit le niveau de remplissage planifié des équipements. La capacité pratique = capacité brute × occ. Ce n'est PAS un facteur appliqué sur la capacité installée — c'est un indicateur de planification pour le magasinier.

ParamètreDéfautExplication
occ_pal — Racks palettes80%Ne jamais viser 100% : il faut de la marge pour les réceptions d'urgence, les palettes en cours de conditionnement, et l'accès aux positions de picking. 80% est la règle professionnelle standard (FEM 9.831).
occ_she — Rayonnage80%Même logique. Au-delà de 85%, les préparateurs ne peuvent plus trouver facilement les emplacements libres lors de mises en stock.
occ_can — Cantilever75%Marge plus large car les items longs sont difficiles à repositionner et nécessitent un dégagement de manœuvre en allée.
occ_flo — Stockage sol75%Les pièces lourdes et encombrantes nécessitent des voies de circulation dégagées en permanence pour le chariot élévateur et/ou le pont roulant.
occ_ded — Zone dédiée100%La zone dédiée est calculée pour correspondre exactement à son besoin. Elle ne sert pas de buffer — laisser à 100%.

ParamètreDéfautFourchetteSignification physique
eff_pal0.450.35 – 0.55Pour 100 m² de zone palette : 45 m² sont des racks, 55 m² sont les allées de manutention (allée principale ≈ 3.5m pour reach truck, allées transversales ≈ 2m). En entrepôt logistique optimisé : 0.50-0.55. En atelier avec contraintes colonnes : 0.35-0.40.
eff_she0.500.40 – 0.60Allées piéton plus étroites (80-100 cm) → meilleure efficacité que les racks palette. 0.50 est valable pour une configuration double-face (rayonnage des 2 côtés d'une allée). Montant si open shelving sans contrainte.
eff_can0.350.30 – 0.45Les allées cantilever doivent être larges pour permettre la mise en place par pont roulant ou chariot à bras latéraux (allée ≥ 4m). D'où une efficacité faible. Réduire à 0.30 si pont roulant unique avec rotation.
eff_flo0.650.55 – 0.70Les emplacements sol sont délimités par des lignes peintes. La circulation se fait dans les allées restantes. 0.65 correspond à une disposition en rangées avec voies de 2.5-3m. Descendre à 0.55 si manutention par grue avec circulation omnidirectionnelle.
